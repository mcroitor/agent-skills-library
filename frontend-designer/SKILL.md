# Frontend Designer (Skeleton CSS)

## Description
A skill for creating lightweight, responsive, and accessible interfaces with Skeleton CSS. It emphasizes simplicity, clarity, and fast delivery.

## When to Use
- Building clean minimalist UI quickly
- Prototyping responsive interfaces
- Using a lightweight CSS framework
- Prioritizing fast page load and low complexity
- Building semantic HTML-first layouts

## Instructions
1. **Understand design goals** - users, brand, and task flow
2. **Sketch layout** - wireframes and information hierarchy
3. **Use Skeleton grid** - 12-column structure for layout
4. **Create reusable components** - buttons, forms, cards, sections
5. **Apply customization** - spacing, colors, typography tokens
6. **Ensure responsiveness** - mobile-first breakpoints
7. **Optimize assets** - compressed images and minimal CSS
8. **Test browsers/accessibility** - keyboard flow and contrast

## Skeleton CSS Highlights
- Very small CSS footprint
- 12-column grid
- Clean typography and form defaults
- Easy to customize
- No JavaScript dependency
- Good baseline for semantic HTML

## Tools and Practices
- HTML5 semantics
- CSS3 (Flexbox/Grid/Custom Properties)
- Skeleton variables and extension patterns
- Responsive design patterns
- Optional SCSS/SASS
- DevTools and Figma workflow
- WCAG 2.1 accessibility basics
