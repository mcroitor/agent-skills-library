# Database Developer (DBA/Database Developer)

## Description
A skill for designing, implementing, and optimizing databases, including schema design, SQL performance tuning, indexing, and operational data reliability.

## When to Use
- Designing new database schemas
- Writing and optimizing SQL queries
- Investigating slow queries
- Building stored procedures and triggers
- Planning backup and recovery
- Migrating data between systems
- Hardening database security
- Monitoring database health

## Instructions
1. **Analyze requirements** - entities, data lifecycle, query patterns
2. **Design schema** - tables, relations, constraints, indexes
3. **Normalize appropriately** - 1NF/2NF/3NF/BCNF with practical trade-offs
4. **Implement DDL** - repeatable migration scripts
5. **Develop queries** - efficient SELECT/INSERT/UPDATE/DELETE
6. **Tune indexing** - support frequent filters and joins
7. **Implement procedures** - encapsulate complex business logic
8. **Secure access** - roles, privileges, encryption
9. **Monitor and improve** - plans, stats, and runtime metrics

## Database Systems
- Relational: PostgreSQL, MySQL, MariaDB, SQL Server, Oracle
- NoSQL: MongoDB, Cassandra, Redis, DynamoDB
- Analytical: ClickHouse, BigQuery, Snowflake
- Graph: Neo4j
- In-memory: Redis, Memcached

## Key Skills
- SQL: joins, aggregations, window functions, CTEs, subqueries
- Indexing: B-tree, Hash, GiST, BRIN
- Query tuning: EXPLAIN/ANALYZE and execution plans
- Transactions: ACID, isolation, locking
- Replication and clustering strategies
- Backup, restore, and PITR planning
- Monitoring, alerting, and capacity planning

## Tools
- pgAdmin, MySQL Workbench, SSMS
- Migration tools: Flyway, Liquibase, Alembic
- Profiling: EXPLAIN, pg_stat_statements
- Monitoring: Prometheus, Grafana
- Backup tools: pg_dump, mysqldump, xtrabackup
