# Skeleton Designer

## Description
A specialized frontend design skill based on Frontend Designer for creating lightweight, responsive interfaces with Skeleton CSS and semantic HTML.

## When to Use
- Building clean minimalist UI quickly
- Prototyping responsive interfaces
- Using a lightweight CSS framework
- Prioritizing fast page load and low complexity
- Building semantic HTML-first layouts

## Instructions
1. **Start from base design** - structure from Frontend Designer skill
2. **Use Skeleton grid** - 12-column layout and simple page rhythm
3. **Create reusable sections** - headers, forms, cards, content blocks
4. **Apply small custom layer** - spacing, colors, typography tokens
5. **Ensure responsiveness** - mobile-first adjustments by breakpoint; if constraints conflict, prioritize accessibility first, then responsiveness, then optimization
6. **Optimize assets** - compress images and keep CSS clean
7. **Test browsers/accessibility** - keyboard flow and contrast
8. **Keep implementation lean** - avoid adding external libraries or frameworks unless they are essential for functionality or performance

## Skeleton CSS Highlights
- Very small CSS footprint
- 12-column grid
- Clean typography and form defaults
- Easy to customize
- No JavaScript dependency
- Good baseline for semantic HTML

## Tools and Practices
- Skeleton CSS
- HTML5 semantics
- CSS3 (Flexbox/Grid/Custom Properties)
- Skeleton variable overrides and extension patterns
- Optional SCSS/SASS
- DevTools and Figma workflow
- WCAG 2.1 accessibility basics
