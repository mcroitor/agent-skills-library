# Bootstrap Designer

## Description
A specialized frontend design skill based on Frontend Designer for building responsive interfaces with Bootstrap components and utility classes.

## Priority Rules
Prioritize in this order when trade-offs conflict:
1. Accessibility
2. Component reusability
3. Responsive behavior
4. Minimal custom CSS

## When to Use
- Delivering UI quickly with a mature component library
- Building admin panels, dashboards, and data-heavy interfaces
- Reusing consistent Bootstrap patterns across pages
- Reducing custom CSS with utility and component classes
- Implementing responsive behavior with Bootstrap breakpoints

## Instructions
1. **Start from base design** - structure from Frontend Designer skill
2. **Use Bootstrap grid** - containers, rows, and columns
3. **Assemble components** - navbar, cards, forms, modals, alerts, tables
4. **Apply utility classes** - spacing, display, typography, alignment
5. **Theme carefully** - variables and overrides with minimal custom CSS
6. **Ensure accessibility** - labels, focus states, ARIA where needed
7. **Optimize bundle** - include only needed Bootstrap assets
8. **Validate responsiveness** - test breakpoints and component behavior

## Input Recovery Rules
- Assume Bootstrap 5.x when version is not specified
- Ask for clarification only when specific Bootstrap components or custom theming significantly affects the approach

## Constraints
- Do not override Bootstrap styles with conflicting custom CSS
- Do not include unused Bootstrap components
- Do not skip accessibility testing with Bootstrap components

## Bootstrap Highlights
- Ready-to-use component library
- Mature 12-column responsive grid
- Strong utility class ecosystem
- Good documentation and ecosystem support
- Fast prototyping with consistent UI results

## Tools and Practices
- Bootstrap 5.x components and utilities
- HTML5 semantics
- CSS3 customization and variable overrides
- Optional SCSS for theme tuning
- DevTools and Figma workflow
- WCAG 2.1 accessibility basics
