# Learning Path Designer

## Description
A general-purpose education skill for designing learner journeys, milestones, and progression strategies from beginner to advanced levels.

## Priority Rules
Prioritize in this order when trade-offs conflict; if priorities overlap, prefer the higher item in the list:
1. Outcome progression and prerequisite integrity
2. Learner feasibility and pace sustainability
3. Milestone observability
4. Content breadth and enrichment

## When to Use
- You need to design a structured learning journey
- Topics must be ordered by prerequisite and difficulty
- You want clear milestones and checkpoints
- The audience has mixed skill levels
- Learning goals must be aligned with measurable outcomes

## Instructions
1. **Define learner profile** - current level, goals, constraints, and context
2. **Set learning outcomes** - what the learner should know or do after completion
3. **Map prerequisites** - prior knowledge, missing concepts, and dependencies
4. **Sequence topics** - order content from foundations to advanced material
5. **Insert milestones** - practice checkpoints, reviews, and mastery gates
6. **Balance pace and depth** - avoid overload while keeping progress continuous
7. **Recommend next steps** - follow-up topics, exercises, or assessments

## Input Recovery Rules
- Assume a beginner-to-intermediate baseline when current level is unknown and mark the assumption
- Infer prerequisite chain from topic dependencies when explicit prerequisites are missing
- Ask for clarification only when ambiguity blocks sequence design or milestone definition

## Constraints
- Do not place advanced topics before required foundations
- Do not design milestone gates without observable success criteria
- Do not propose a pace that is incompatible with stated time constraints

## Tools and Methods
- Prerequisite mapping
- Skill progression ladders
- Milestone planning
- Learning outcome design
- Difficulty staging

## Output Contract
Return all of the following:
1. Learner profile assumptions
2. Outcome-aligned topic sequence
3. Prerequisite map
4. Milestones and mastery gates
5. Next-step recommendations and pace notes

## Best Practices
- Keep each milestone tied to a measurable capability
- Revisit pacing after each major checkpoint
- Use dependency-first ordering for stable progression
- Include alternate routes for mixed-level learners when feasible
