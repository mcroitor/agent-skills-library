# Test Scenario Writer (QA/Test Design)

## Description
A skill for creating comprehensive test scenarios and test cases from requirements, including happy paths, edge cases, and risk-based coverage.

## When to Use
- Building test plans for new features
- Estimating and improving test coverage
- Capturing edge and negative cases
- Preparing regression suites
- Formalizing exploratory test charters
- Defining acceptance criteria for QA

## Instructions
1. **Review requirements** - specs, stories, and acceptance criteria
2. **Map user flows** - primary and alternative paths
3. **Identify edge cases** - invalid input and boundary values
4. **Prepare test data** - valid, invalid, and stateful data sets
5. **Plan coverage** - prioritize by risk and business impact
6. **Write test cases** - clear steps and expected outcomes
7. **Document consistently** - reusable, readable case format
8. **Prioritize execution** - critical, high, medium, low
9. **Review quality** - completeness, traceability, maintainability

## Test Case Structure
- Test ID and title
- Preconditions
- Test steps
- Expected result
- Actual result
- Status (Pass/Fail)
- Notes and linked defects

## Methods and Approaches
- Black-box testing
- Boundary Value Analysis
- Equivalence Partitioning
- Decision Table Testing
- State Transition Testing
- Exploratory Testing
- Risk-based Testing
- Regression strategy and maintenance
